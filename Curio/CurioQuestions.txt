1.	Offline ve periodik dispatch için farklı servisler hizmet vermeye devam edecek. 
2.	Offline servisi 
	a.	Authenticated olmayacak (yani sessioncode üst seviyede olmayacak. row dataların içinde ilgili sessioncode’ları olacak)
	b.	ApiKey ile kendisi authentication’unu yapacak 
	c.	Gönderilen datayı session’ların timeout olma sürelerini kontrol ederek gerektiğinde yeni session create ederek reddetmeden işleyecek. 
	d.	En son ürettiği sessionCode’u client’a dönecek. Client sonraki taleplerde bu sessioncode’u kullanabilecek:
		i.	Sonraki online ve periodik işlemler için eğer bu code timeout olmuş ise unauthenticated dönülecek, ve client yeni bir session oluşturacak
		ii.	Timeout olmamış ise bu session’dan devam edilebilecek. 
		iii.	Sonraki offline işlemler için ise sorun yok, güncel sessioncode server tarafından oluşturularak client’a dönülecek. 
3.	Periodik servisi
	a.	Authenticated servis
	b.	Sessioncode olarak gönderilen session’a dahil edilecek buradaki datalar
	c.	Eğer timeout olmuş ise unauthenticated dönülerek client’in yeniden göndermesi beklenecek 
4.	Periodik datalar cihazın offline’a düşmesi durumunda offline’a çevrilip yani sessionCode’ları set edilerek offline servisine gönderilecek. ( sessiontimeout alınmaması ve hangi session’a dahil edileceğinin bilinmesi için )
5.	Online Create session servisi sessioncode’u client’tan alır hale getirilecek. Eğer cihaz göndermezse server üretecek.  (SessionCode timebased uuid olmalı. )
